4.计算机组成原理
	[1] 硬件
		(1)冯﹒诺依曼式体系结构
		理论要点
			1、计算机硬件设备由存储器、运算器、控制器、输入设备和输出设备5部分组成。				1) 输入设备和输出设备 (I/O设备)
					输入设备: 键盘、鼠标、游戏杆、麦克风、摄像头、触控板、扫描仪 ... 
					输出设备: 显示器、扬声器、投影仪、打印机 ..... 
				2) 存储器 : 主存，一类记忆设备。存放了机器指令和操作的数据。
				3) 运算器 :  ALU可以执行算术运算(包括加减乘数等基本运算及其附加运算)和
										逻辑运算(包括移位、逻辑测试或两个值比较)
				4) 控制器 : 从系统通电开始，直到系统断电	取指  ->  译码 -> 执行
				
			2、存储程序思想——指令和数据不加区别混合存储在同一个存储器中，数据和程序在内存中是没有区别的,
				它们都是内存中的数据
			3、计算机的数制: 二进制 
			...
		(2) 典型计算机系统组成 
			1) CPU  
				运算器 ALU  
				控制器 
				寄存器 register : 是CPU内部用来存放数据的一些小型存储区域，用来暂时存放参与运算的数据和运算结果。
								  速度快，数目少。特殊的寄存器叫PC（程序计数器），在任何时刻PC都指向主存中的某条机器指令的地址。
			2) I/O设备
			3) 主存
			4) 总线(BUS) ：是计算机各种功能部件之间传送信息的公共通信干线
				数据总线DB( Data Bus): 传输数据
				地址总线AB(Address Bus): 传输地址
				控制总线CB(Control Bus): 传输控制信号(读/写, 片选, 中断, 复位信号...)
				
				USB总线、I2C总线、SPI总线 、I2S总线、PCI总线 ...
				
		(3) 存储器的种类  
			1) 主存储器: 内存, 是计算机的工作存储器。程序，待处理的数据和处理的结果都存储在内存中。
					内存读写速度比较快，但只能在加电时存储数据
			
			2) 寄存器 : 是CPU内部用来存放数据的一些小型存储区域
			
			3) 外存储器: 外存储器来长期保存数据的大容量存储器，目前多为磁存储器或光存储器。读写速度慢于内存，可以在不加电的情况下长期保存数据。
			
			4) 高速缓存 ：CPU高速缓存（英语：CPU Cache，简称缓存）是用于减少处理器访问内存所需平均时间的部件。

		(3) Linux系统简介
			[1] 起源  
				1969年，AT&T 公司的贝尔实验室与MIT合作开发的 Unix
				旨在于创建一个用于大型、并行、多用户的操作系统。
				
				Unix的版本主要两个： 
					AT&T System V   -- 就是俗称的“系统5”
					Berkley Software Distribution (BSD) -- 加州大学伯克利分校伯克利软件中心，简称"伯克利"
				
				Linus Torvals 	1991年，芬兰赫尔辛基大学的学生	
				为了能在家里的PC机上使用与学校一样的操作系统，开始编写了类UNIX
				参考  minix(类Unix 教学目的)
				
				1991.8.25，Linus就在comp.os.minix新闻组中首次发布了一个Linux内核的公共版本。
				最初Linus编写的程序只适用于Intel386处理器，且不能移植。由于人们的鼓励，继续编写可移植的Linux系统。
				Linux = Linus + Unix 
				
				Linux 基金会

5.Linux 操作系统   
	[1] Linux的文件系统	
		(1) 定义
			操作系统中负责管理和存储文件信息的软件机构称为文件管理系统，简称文件系统。
		(2) Linux文件系统结构
			1) 目录在分区下
				微软Windows操作系统将硬盘上的几个分区，用C：、D：、E：、F：等符号标识。
				存取文件时一定要清楚存放在哪个磁盘的哪个目录下。
			2) 分区在目录下
				Linux的文件组织模式犹如一颗倒置的树,所有存储设备作为这颗树的一个子目录。 
				Linux 文件系统的起点 ：根目录 "/"
		(3) 文件系统层次结构标准FHS
			为了规范文件目录命名和存放标准，Linux基金会颁发了FHS(Filesystem Hierarchy Standard)。
			
			/		Linux文件系统"根目录"
			/bin	存放系统中最常用的可执行文件（二进制）
			/boot	存放Linux内核和系统启动文件，包括Grub、lilo启动器程序
			/dev	存放所有设备文件，包括硬盘、分区、键盘、鼠标、USB、tty等
			/etc	存放系统的所有配置文件，例如passwd存放用户账户信息，hostname存放主机名等
			/home	用户主目录的默认位置
			/lib	存放共享的库文件，包含许多被/bin和/sbin中程序使用的库文件
			/root	根用户（超级用户）的主目录
			......
		(4) Linux 文件路径  
			在Linux文件系统中查找一个文件，只要确定文件名和路径，就可以唯一确定这个文件
			1) 绝对路径 : 指文件在文件系统中的准确位置。通常在本地主机上，以根目录为起点。
					/boot/grub
				根目录   分隔符  	
					/usr/games/gnomine  	扫雷程序
			
			2) 相对路径 :指相对于用户当前位置的一个文件或目录的相对位置，以当前目录为起点。
				.			表示当前目录
				..			表示上级目录
				../..		表示上上级目录
				
				eg： ../../share 
					../../usr/games/gnomine
					
	[2] 终端 Terminal  
		1) 什么是终端：用户登录、执行指令或程序 
			Windows 前身 为 Dos系统 ，早期的操作系统无图形界面，为了实现人机交互：
				用户在终端上输入一些系统预定义的一些指令，由终端上的特定应用程序进行解析 
				并且执行相关操作，最后将结果显示在终端上。
					运行 cmd  或 powershell  
						Dos指令
							ping www.baidu.com 
							shutdown -s -t time(秒)
							shutdown -a 
		2) Linux终端操作
			a. 打开终端  
				ctrl + alt + t 或 ctrl + shift + n
			b. 放大终端字体 
				ctrl + shift + '+' 
			c. 缩小终端字体
				ctrl + '-'
			d. 关闭终端
				alt + F4 
				exit -- 用户退出登录(如果当前终端只有当前用户时，在退出登录的同时会关闭终端)
			
		3) 命令提示符
			命令提示符标识了命令行的开始。用户在提示符后面输入一条命令并按【Enter】键，完成向系统提交指令。
		格式:
			fs@ubuntu:~$
			root@ubuntu:/home/fs#
			用户名@主机名:当前路径$
			
			用户类型: 
				$	普通用户
				#	超级用户 (root 相当于管理员，拥有整个系统的最高权限)
				
		4) 用户主目录(家目录)
			Linux系统在创建新用户的时会为每一个用户创建一个属于该用的主目录(家目录)；
		如果没有权限，其他用户无法访问。
			位置： 
				普通用户： 默认在 /home 创建与用户名同名的目录作为 主目录  
						fs =>  /home/fs  
						zhangsan  => /home/zhangsan 
				超级用户:  
						/root	根用户（超级用户）的主目录	
			
			注意：对于当前用户，可以使用 "~" 表示主目录的路径
